Spread operator

The Spread Operator (...) - Unpacking Elements

The spread operator "unpacks" or "expands" an iterable (like an array, string, or object) into its individual elements. Think of it as taking a packed suitcase and laying all its contents out on the bed. You use it where you need individual elements, such as in array literals, object literals, or function calls.

Example: Merging arrays
const frontEnd = ['React', 'Vue', 'Svelte'];
const backEnd = ['Node.js', 'Express', 'NestJS'];

// Combining two arrays into a new one
const fullStack = [...frontEnd, ...backEnd];
console.log(fullStack); 
// Output: ['React', 'Vue', 'Svelte', 'Node.js', 'Express', 'NestJS']

// Adding an element to the beginning of an array immutably
const withAngular = ['Angular', ...frontEnd];
console.log(withAngular); 
// Output: ['Angular', 'React', 'Vue', 'Svelte']

Shallow copy Behavior

Before ES6, copying an array required methods like slice(). The spread operator makes this much more intuitive. This is crucial for avoiding accidental modifications to the original data.

Example with Array
// Before ES6
const original = ['a', 'b', 'c'];
const sliceCpy = original.slice(0,original.length);
console.log(original); // Output: ['a', 'b', 'c'] (The original is safe!)
console.log(sliceCpy); // Output: ['a', 'b', 'c', 'd']

// After ES6
const copy = [...original];
copy.push('d'); // Modify the copy
console.log(original); // Output: ['a', 'b', 'c'] (The original is safe!)
console.log(copy);     // Output: ['a', 'b', 'c', 'd']

The Rest Parameter (...) - Gathering Elements

You can only use it at the end of a function's parameter list.

The rest parameter does the exact opposite. It "collects" an indefinite number of arguments and bundles them into a single array. Think of it as gathering scattered items from a table and putting them all into one bag.

Example: Function calling multiple numbers as an argument
// A function that calculates the sum of all numbers passed to it.
function sum(...numbers) {
  // 'numbers' is now a real array.
  return numbers.reduce((total, current) => total + current, 0);
}

console.log(sum(1, 2, 3));              // Output: 6
console.log(sum(10, 20, 30, 40, 50)); // Output: 150
console.log(sum());                     // Output: 0
Destructuring an Array
const leaderboard = ['Alice', 'Bob', 'Charlie', 'David', 'Eve'];

// Get the winner, runner-up, and everyone else.
const [winner, runnerUp, ...everyoneElse] = leaderboard;

console.log(winner);        // Output: 'Alice'
console.log(runnerUp);      // Output: 'Bob'
console.log(everyoneElse);  // Output: ['Charlie', 'David', 'Eve']
Creating More Flexible and Readable Functions
// A logging function that takes a required category and then any number of messages.
function logMessage(category, ...messages) {
  const timestamp = new Date().toISOString();
  console.log(`[${timestamp}] [${category.toUpperCase()}]`);
  
  messages.forEach(message => {
    console.log(`  - ${message}`);
  });
}

logMessage('INFO', 'User logged in', 'Accessed dashboard page');
// Output:
// [2025-09-25T....] [INFO]
//   - User logged in
//   - Accessed dashboard page

logMessage('ERROR', 'Failed to connect to database');
// Output:
// [2025-09-25T....] [ERROR]
//   - Failed to connect to database
